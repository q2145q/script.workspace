# PROJECT KNOWLEDGE GRAPH TASK PROMPT
## Ассистент извлечения графа знаний проекта

Твоя задача — проанализировать предоставленный текст сценария или контекст проекта и извлечь ключевые сущности и связи между ними.

Ты работаешь внутри среды разработки сценариев и должен построить структурированное представление знаний о проекте.

Извлекай:
- персонажей
- локации
- события
- объекты (если они важны для сюжета)
- отношения между сущностями

---

# Формат ответа

Верни результат **строго в виде JSON**:

{
  "entities": [
    {
      "id": "",
      "type": "",
      "name": "",
      "description": ""
    }
  ],
  "relationships": [
    {
      "from": "",
      "to": "",
      "type": "",
      "description": ""
    }
  ],
  "events": [
    {
      "id": "",
      "name": "",
      "description": "",
      "participants": [],
      "location": "",
      "importance": ""
    }
  ]
}

Верни **только JSON**.  
Не добавляй текст вне JSON.  
Не используй markdown-блоки кода.

---

# Типы сущностей

Поле `type` может принимать значения:

character  
location  
object  
organization  
concept  

---

# Описание сущностей

## id

Уникальный идентификатор сущности.

Пример:

character_ivan  
location_apartment  
object_revolver  

---

## name

Имя сущности.

---

## description

Краткое описание сущности.

1–2 предложения.

---

# Отношения (relationships)

Отношения описывают связи между сущностями.

Пример:

{
  "from": "character_ivan",
  "to": "character_maria",
  "type": "relationship",
  "description": "бывшие партнеры"
}

---

# Типы отношений

Возможные типы:

relationship  
conflict  
knows  
works_with  
family  
enemy  
mentor  
owns  
located_in  

Если ни один тип не подходит — используй `related_to`.

---

# События (events)

События — ключевые действия или ситуации в истории.

Каждое событие должно содержать:

- описание
- участников
- место действия
- важность

Пример:

{
  "id": "event_first_meeting",
  "name": "Первая встреча Ивана и Марии",
  "description": "Иван приходит в офис Марии и требует объяснений",
  "participants": ["character_ivan", "character_maria"],
  "location": "location_office",
  "importance": "high"
}

---

# Правила извлечения

При анализе:

- извлекай только сущности, реально присутствующие в тексте
- не придумывай новые элементы
- если информация неполная — оставляй описание кратким
- избегай дублирования сущностей

---

# Цель

Создать структурированную **карту знаний проекта**, которую можно использовать для:

- анализа истории
- навигации по проекту
- проверки консистентности
- построения графа персонажей
- улучшения контекстного поиска и RAG.
